<objective>
## Goal
Add a `manager` role between `admin` and `staff`, giving managers elevated access to production, timecards, and customer data — without the ability to manage billing, subscriptions, employees, or tenant settings.

## Purpose
Owners need to delegate daily operations to a trusted manager without handing over full admin access. Currently, everything is binary: admin (full control) or staff (view only).

## Access Matrix

| Capability | Admin | Manager | Staff |
|---|---|---|---|
| Billing / subscription / onboarding | ✓ | ✗ | ✗ |
| Invite / remove / promote employees | ✓ | ✗ | ✗ |
| Pricing settings | ✓ | ✗ | ✗ |
| Customers — full CRUD | ✓ | ✓ | read |
| Quotes — create / edit / send | ✓ | ✓ | view |
| Orders — approve / manage | ✓ | ✓ | view |
| Invoices — create / send | ✓ | ✓ | ✗ |
| Production board | ✓ | ✓ | ✓ |
| Timeclock admin dashboard + editing | ✓ | ✓ | own only |
| Timeclock approval | ✓ | ✓ | ✗ |
| Timeclock reports + export | ✓ | ✓ | ✗ |
| Staff profiles (kiosk) | ✓ | ✓ | ✗ |
| Product catalog | ✓ | ✓ | ✗ |
| Preview Portal | ✓ | ✗ | ✗ |

## Output
- `manager` is a valid role in `public.user_role` enum
- Admins can assign manager role via /dashboard/settings/employees/[id]/edit
- All route guards updated per access matrix
- `isElevated` helper (admin or manager) used consistently throughout
</objective>

    Also update `requireAdmin()` — keep it strict (admin-only). Add a new helper:
    ```ts
    async function requireElevated() {
      const supabase = await createClient()
      const { data: { user } } = await supabase.auth.getUser()
      if (!user) throw new Error('Not authenticated')
      const { data: profile } = await supabase.from('users').select('role').eq('id', user.id).single()
      if (!['admin', 'manager'].includes(profile?.role ?? '')) throw new Error('Manager access required')
      return { user, role: profile!.role }
    }
    ```

    Do NOT use requireElevated for inviteEmployee, removeEmployee — those stay admin-only.
  </action>
  <verify>TypeScript compiles: npx tsc --noEmit</verify>
