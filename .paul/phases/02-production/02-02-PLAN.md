<objective>
## Goal
Add a file upload system — staff can attach design files and proofs to an order. Files are uploaded directly to S3 via presigned URLs; metadata is persisted in a `files` table in Supabase.

## Purpose
Print shops work with large design files (PDF, AI, EPS, TIFF). Uploading via the server would be slow and wasteful; S3 presigned URLs let the client PUT directly to S3, with the server only generating the URL and recording the result. This is Phase 2's "Files" deliverable.

## Output
- `supabase/migrations/20260327000002_files.sql` — files table + RLS
- `src/lib/actions/files.ts` — `createPresignedUploadUrl`, `recordUploadedFile`, `deleteFile`
- `src/components/file-uploader.tsx` — `'use client'` upload component using presigned URL
- `src/app/dashboard/orders/[id]/page.tsx` — files section added to order detail
</objective>

    **Migration (`20260327000002_files.sql`):**
    Create `public.files` table:
    ```sql
    create table public.files (
      id uuid primary key default uuid_generate_v4(),
      order_id uuid references public.orders(id) on delete cascade not null,
      name text not null,           -- original filename
      s3_key text not null,         -- S3 object key: uploads/{order_id}/{uuid}-{name}
      content_type text not null,
      size_bytes bigint not null,
      uploaded_by uuid references auth.users(id),
      created_at timestamptz default now()
    );
    ```
    Enable RLS. Policies:
    - SELECT: `public.is_authenticated()`
    - INSERT: `public.is_authenticated()`
    - DELETE: `public.is_admin()`

    **Server actions (`src/lib/actions/files.ts`):**
    Mark `'use server'`. Import `@aws-sdk/client-s3` and `@aws-sdk/s3-request-presigner`.

    Read env vars: `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_REGION`, `S3_BUCKET_NAME` from `process.env`.

    Content-type allowlist:
    ```ts
    const ALLOWED_TYPES = [
      'application/pdf',
      'image/png',
      'image/jpeg',
      'image/gif',
      'image/tiff',
      'application/postscript',          // .ai / .eps
      'application/illustrator',
    ]
    ```

    `createPresignedUploadUrl(orderId: string, filename: string, contentType: string, sizeBytes: number)`:
    - Verify user is authenticated via Supabase
    - Reject if contentType not in ALLOWED_TYPES — throw Error('File type not allowed')
    - Reject if sizeBytes > 100 * 1024 * 1024 (100 MB) — throw Error('File too large (max 100 MB)')
    - Generate s3Key: `uploads/${orderId}/${crypto.randomUUID()}-${filename}`
    - Create S3Client with credentials from env
    - Use `getSignedUrl` with `PutObjectCommand` — expiry 300 seconds, ContentType set
    - Return `{ presignedUrl, s3Key }`

    `recordUploadedFile(orderId: string, name: string, s3Key: string, contentType: string, sizeBytes: number)`:
    - Insert into `public.files` table
    - `revalidatePath(\`/dashboard/orders/${orderId}\`)`
    - Return the inserted file row

    `deleteFile(fileId: string)`:
    - Fetch file row to get s3Key and order_id
    - Delete from S3 using `DeleteObjectCommand`
    - Delete from `public.files`
    - `revalidatePath(\`/dashboard/orders/${file.order_id}\`)`

    Install packages: `@aws-sdk/client-s3`, `@aws-sdk/s3-request-presigner`
    Run: `npm install @aws-sdk/client-s3 @aws-sdk/s3-request-presigner` in apps/print-shop-crm.

    Also append to `.env.local.example` (do not write actual values — placeholders only):
    ```
    AWS_ACCESS_KEY_ID=your-key-here
    AWS_SECRET_ACCESS_KEY=your-secret-here
    AWS_REGION=us-east-1
    S3_BUCKET_NAME=your-bucket-name
    ```
  </action>

    **`src/components/file-uploader.tsx`** — `'use client'` component:
    Props: `{ orderId: string; onUploaded: () => void }`

    State: `uploading: boolean`, `error: string | null`, `progress: number | null`

    Flow on file select:
    1. Validate file type client-side against the same allowlist (MIME type check)
    2. Call `createPresignedUploadUrl(orderId, file.name, file.type, file.size)` — this is a server action, call it directly
    3. PUT the file directly to the returned `presignedUrl` using `fetch` with `method: 'PUT'`, `body: file`, `headers: { 'Content-Type': file.type }`
    4. On 200, call `recordUploadedFile(orderId, file.name, s3Key, file.type, file.size)`
    5. Call `onUploaded()` to trigger parent refresh
    6. On error at any step, set error state and display it

    UI: file `<input type="file" accept=".pdf,.png,.jpg,.jpeg,.gif,.tiff,.ai,.eps" />`, upload button, spinner during upload, error message below.

    **`src/app/dashboard/orders/[id]/page.tsx`** — add files section:
    - Add a second Supabase query: `supabase.from('files').select('*').eq('order_id', params.id).order('created_at', { ascending: false })`
    - Add a `<FileUploader>` section below the order details — wrap in a client-component boundary: since the page is a server component, use `FileUploader` directly (it's already 'use client')
    - Add a files list below FileUploader showing: filename, size (format as KB/MB), upload date, and a delete button (form action calling `deleteFile`)
    - The delete button is an admin-only action — check the user's role fetched from `public.users` and only render delete if `role === 'admin'`
    - For the `onUploaded` refresh: pass `router.refresh` — but since the page is a server component, create a thin `'use client'` wrapper `OrderFilesPanel` that holds the FileUploader and handles the refresh via `useRouter`

    Format file sizes: < 1024 bytes → "X B", < 1024^2 → "X KB", else → "X.X MB"
