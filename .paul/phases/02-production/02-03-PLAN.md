<objective>
## Goal
Wire n8n Cloud into the CRM so that key lifecycle events (quote sent, order approved, job ready for pickup) automatically trigger email and SMS notifications via n8n — and scaffold the inbound n8n webhook route for Phase 3 use.

## Purpose
Staff currently have no automated way to notify customers or themselves when a job moves through the pipeline. This plan adds that layer: the CRM fires outbound events to n8n Cloud webhook URLs; n8n handles delivery (email via n8n native, SMS via Twilio). The inbound webhook route is scaffolded now so Phase 3 (Stripe payment confirmation) can use it without new infrastructure work.

## Output
- `src/lib/n8n.ts` — `notifyN8n(event, payload)` utility with shared secret header
- `src/lib/actions/quotes.ts` — fires `quote.sent` when quote status → 'sent'
- `src/lib/actions/orders.ts` — fires `order.approved` when order status → 'approved'
- `src/lib/actions/jobs.ts` — fires `job.ready_for_pickup` when stage → 'ready_for_pickup'
- `src/app/api/webhooks/n8n/route.ts` — POST route, shared secret validation, event dispatch scaffold
- `.env.local.example` — n8n webhook URL vars + shared secret placeholder
</objective>

## AC-1: Quote Sent Notification
```gherkin
Given a quote exists in 'draft' status
When an admin marks the quote as 'sent'
Then a POST is fired to the N8N_WEBHOOK_QUOTE_SENT URL with the quote ID and customer details
```

## AC-2: Order Approved Notification
```gherkin
Given an order exists in 'pending' status
When an admin approves the order
Then a POST is fired to the N8N_WEBHOOK_ORDER_APPROVED URL with the order ID and customer details
```

## AC-3: Ready for Pickup Notification
```gherkin
Given a job is in stage 'finishing'
When staff advances the job to 'ready_for_pickup'
Then a POST is fired to the N8N_WEBHOOK_JOB_READY URL with the job ID, order ID, and customer contact info (for email + SMS)
```

## AC-4: Notification Failures Are Non-Blocking
```gherkin
Given a notification webhook call fails (n8n unreachable, misconfigured URL, or missing env var)
When the underlying action (quote status update, order approval, job stage advance) completes successfully
Then the action still succeeds — notification failure is logged to console but never throws to the caller
```

## AC-5: Inbound Webhook Route Validates Shared Secret
```gherkin
Given a POST arrives at /api/webhooks/n8n
When the x-n8n-secret header does not match N8N_WEBHOOK_SECRET
Then the route returns 401
When the header matches
Then the route returns 200 and logs the received event
```

  <name>Task 1: n8n notification utility + env vars</name>

  <action>
    **`src/lib/n8n.ts`** — notification utility (NOT 'use server' — plain module, imported by server actions):

    Export `notifyN8n(webhookUrl: string | undefined, event: string, payload: Record<string, unknown>): Promise<void>`

    Logic:
    - If `webhookUrl` is undefined or empty, return immediately (env var not configured — graceful no-op)
    - POST to `webhookUrl` with:
      - `Content-Type: application/json`
      - `x-n8n-secret: process.env.N8N_WEBHOOK_SECRET ?? ''` header
      - JSON body: `{ event, ...payload, timestamp: new Date().toISOString() }`
    - Wrap the fetch in try/catch — on any error, `console.error('[n8n]', event, err.message)` and return without throwing
    - Never throw — callers must not fail because n8n is unavailable

    Also export three typed helper functions that call `notifyN8n` with the right URL env var:

    `notifyQuoteSent(quoteId: string, customerEmail: string, customerName: string)`:
    - Calls `notifyN8n(process.env.N8N_WEBHOOK_QUOTE_SENT, 'quote.sent', { quoteId, customerEmail, customerName })`

    `notifyOrderApproved(orderId: string, customerEmail: string, customerName: string)`:
    - Calls `notifyN8n(process.env.N8N_WEBHOOK_ORDER_APPROVED, 'order.approved', { orderId, customerEmail, customerName })`

    `notifyJobReady(jobId: string, orderId: string, customerEmail: string, customerName: string, customerPhone: string | null)`:
    - Calls `notifyN8n(process.env.N8N_WEBHOOK_JOB_READY, 'job.ready_for_pickup', { jobId, orderId, customerEmail, customerName, customerPhone })`

    **`.env.local.example`** — append (placeholder values only, never real credentials):
    ```
    N8N_WEBHOOK_SECRET=your-shared-secret-here
    N8N_WEBHOOK_QUOTE_SENT=https://your-n8n-instance.com/webhook/quote-sent
    N8N_WEBHOOK_ORDER_APPROVED=https://your-n8n-instance.com/webhook/order-approved
    N8N_WEBHOOK_JOB_READY=https://your-n8n-instance.com/webhook/job-ready
    ```
  </action>

  <action>
    Create `src/app/api/webhooks/n8n/route.ts` — Next.js route handler (no 'use server' needed for route handlers).

    Export `POST(request: Request)`:
    1. Read `x-n8n-secret` header from request
    2. Compare to `process.env.N8N_WEBHOOK_SECRET` — if missing or mismatch, return `Response.json({ error: 'Unauthorized' }, { status: 401 })`
    3. Parse request body as JSON: `const body = await request.json()`
    4. Extract `event` from body (string)
    5. Log: `console.log('[n8n webhook]', event, body)`
    6. Switch on `event`:
       - `'payment.confirmed'` → log "Payment confirmed event received (Phase 3 handler pending)" and return 200
       - default → log "Unhandled event: [event]" and return 200
    7. Return `Response.json({ received: true }, { status: 200 })`

    This is a scaffold — Phase 3 will add actual payment confirmation logic inside the `'payment.confirmed'` case.

    Do NOT import createClient or do any DB work — this is a pure scaffold with secret validation and event logging only.
