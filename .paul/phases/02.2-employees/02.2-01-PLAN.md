<objective>
## Goal
Build an admin employee management page where admins can view all staff in their tenant and edit employee names and roles — without needing Supabase access.

## Purpose
Currently the only way to add or edit staff is via Supabase SQL editor. This is a blocker for real shop owners onboarding employees. This plan adds the read + edit half; Plan 02 adds invite.

## Output
- `/dashboard/settings/employees` — list of all tenant employees with edit links
- `/dashboard/settings/employees/[id]/edit` — edit employee name and role
- `src/lib/actions/employees.ts` — `updateEmployee` server action
- Nav sidebar updated to surface the Employees page
</objective>

<acceptance_criteria>

## AC-1: Employee list visible to admin
```gherkin
Given an admin is logged in
When they navigate to /dashboard/settings/employees
Then they see a table listing all employees in their tenant
  with each row showing: name (or email fallback), email, role badge, and an Edit link
```

## AC-2: Non-admin cannot access employee management
```gherkin
Given a staff user is logged in
When they navigate to /dashboard/settings/employees
Then they are redirected to /dashboard
```

## AC-3: Admin can edit employee name
```gherkin
Given an admin is on the edit page for an employee
When they change the name field and submit
Then the employee's name is updated in public.users
  and they are redirected back to the employee list
```

## AC-4: Admin can change employee role
```gherkin
Given an admin is on the edit page for an employee
When they change the role dropdown (staff ↔ admin) and submit
Then the employee's role is updated in public.users
  and they are redirected back to the employee list
```

## AC-5: Admin cannot remove their own admin role
```gherkin
Given an admin is editing their own record
When they view the role dropdown
Then the role field is disabled (cannot demote themselves)
```

</acceptance_criteria>

<task type="auto">
  <name>Task 1: updateEmployee server action</name>
  <files>src/lib/actions/employees.ts</files>
  <action>
    Create new file with one exported server action: `updateEmployee(formData: FormData)`.

    - Extract `id`, `name`, `role` from formData
    - Call `requireAdmin()` pattern (same as timeclock.ts lines 71-87: get user, check role = 'admin')
    - Validate `role` is 'admin' or 'staff' — reject any other value
    - Guard: if `id === currentUser.id` and `role !== 'admin'`, throw Error('Cannot remove your own admin role')
    - UPDATE public.users SET name = name, role = role WHERE id = id AND tenant_id = get_my_tenant_id() (use supabase authenticated client — admin RLS policy covers this)
    - After update: `redirect('/dashboard/settings/employees')`
    - Import pattern: follow timeclock.ts exactly (createClient, requireAdmin, redirect, revalidatePath)
  </action>
  <verify>TypeScript compiles without errors: npx tsc --noEmit</verify>
  <done>AC-3 and AC-4 satisfied: server action updates name and role, redirects on success</done>
</task>

<task type="auto">
  <name>Task 2: Employee list page + edit page</name>
  <files>
    src/app/dashboard/settings/employees/page.tsx,
    src/app/dashboard/settings/employees/[id]/edit/page.tsx
  </files>
  <action>
    **List page** (`employees/page.tsx`):
    - Server component (no 'use client')
    - Get user + profile; if profile.role !== 'admin' → redirect('/dashboard')
    - Query: `supabase.from('users').select('id, email, name, role, created_at').eq('tenant_id', tenantId).order('created_at')` — use service client (createServiceClient) since admin reads own tenant users via standard RLS policy
    - Wait — use authenticated client with `get_my_tenant_id()` — the "Admins can view tenant users" RLS policy handles this
    - Render a table: columns = Employee (name || email), Email, Role (badge), Since, Actions (Edit link)
    - Role badge: admin = dark gray pill, staff = light gray pill
    - Edit link → `/dashboard/settings/employees/[id]/edit`
    - Match visual style of timeclock admin page (bg-white rounded-xl border, text-sm table)

    **Edit page** (`employees/[id]/edit/page.tsx`):
    - Server component
    - Admin-only guard (same pattern)
    - Fetch employee: `supabase.from('users').select('id, email, name, role').eq('id', params.id).eq('tenant_id', tenantId).single()`
    - If not found → redirect to list
    - Get current user id (to disable role field if editing self)
    - Form with action={updateEmployee}:
      - Hidden input: name="id" value={employee.id}
      - Text input: name="name" defaultValue={employee.name ?? ''} placeholder="Full name"
      - Select: name="role" defaultValue={employee.role}
        - Options: staff, admin
        - disabled={employee.id === currentUserId} (AC-5)
      - Submit button + cancel link back to list
    - Match visual style of timeclock admin edit page
  </action>
  <verify>
