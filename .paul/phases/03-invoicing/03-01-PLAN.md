<objective>
## Goal
Add invoice generation to the CRM — admin can generate an invoice from a completed order, view all invoices with status, and see per-invoice detail. No payment processing yet (that's 03-02); this plan lays the data model and UI.

## Purpose
Phase 3 is the "order-to-cash" phase. Before wiring Stripe, the app needs an invoice entity with its own lifecycle (draft → sent → paid), a way to create one from an order, and a management UI. This plan delivers that foundation so 03-02 can add the payment layer on top without restructuring.

## Output
- `supabase/migrations/20260327000003_invoices.sql` — invoices table + RLS
- `src/lib/types.ts` — `InvoiceStatus` type added
- `src/lib/actions/invoices.ts` — `createInvoice`, `markInvoiceSent`, `deleteInvoice`
- `src/app/dashboard/orders/[id]/page.tsx` — invoice section: generate button or status card

- `src/app/dashboard/invoices/[id]/page.tsx` — invoice detail page
- `src/components/nav-sidebar.tsx` — Invoices nav item added

  <name>Task 1: Invoices migration + types + server actions</name>
  <files>

  <action>
    **Migration (`20260327000003_invoices.sql`):**
    ```sql
    create table public.invoices (
      id uuid primary key default uuid_generate_v4(),
      order_id uuid references public.orders(id) on delete cascade not null unique,
      amount numeric(10,2) not null,
      status text not null default 'draft'
        check (status in ('draft', 'sent', 'paid')),
      due_date date,
      stripe_payment_intent_id text,
      stripe_payment_link_url text,
      created_by uuid references auth.users(id),
      created_at timestamptz default now(),
      updated_at timestamptz default now()
    );
    ```
    Note: `order_id` has a `unique` constraint — one invoice per order enforced at DB level.

    Enable RLS:
    - SELECT: `public.is_authenticated()`
    - INSERT: `public.is_admin()`
    - UPDATE: `public.is_admin()`
    - DELETE: `public.is_admin()`

    Add updated_at trigger:
    ```sql
    create trigger invoices_updated_at
      before update on public.invoices
      for each row execute procedure public.set_updated_at();
    ```

    **`src/lib/types.ts`** — append at end of file:
    ```ts
    export const INVOICE_STATUS_SEQUENCE = ['draft', 'sent', 'paid'] as const
    export type InvoiceStatus = (typeof INVOICE_STATUS_SEQUENCE)[number]
    ```

    **`src/lib/actions/invoices.ts`** — mark `'use server'`:

    `createInvoice(orderId: string)`:
    - Verify user is authenticated AND admin (check `public.users` role)
    - Fetch order: `id`, `total`, `customer_id` from `public.orders`
    - If order not found, throw `'Order not found'`
    - Check no existing invoice: query `public.invoices` where `order_id = orderId` — if exists, throw `'Invoice already exists for this order'`
    - Insert invoice: `{ order_id: orderId, amount: order.total, status: 'draft', created_by: user.id }`
    - `revalidatePath('/dashboard/orders/' + orderId)`
    - `revalidatePath('/dashboard/invoices')`
    - Return inserted invoice row (select after insert)

    `markInvoiceSent(invoiceId: string)`:
    - Verify authenticated + admin
    - Update `public.invoices` set `status = 'sent'` where `id = invoiceId` and `status = 'draft'`
    - Fetch `order_id` from invoice row (needed for revalidation)
    - `revalidatePath('/dashboard/invoices/' + invoiceId)`
    - `revalidatePath('/dashboard/orders/' + invoice.order_id)`
    - `redirect('/dashboard/invoices/' + invoiceId)`

    `deleteInvoice(invoiceId: string)`:
    - Verify authenticated + admin
    - Fetch invoice to get `order_id`
    - Only allow delete if `status = 'draft'` — throw `'Only draft invoices can be deleted'` otherwise
    - Delete from `public.invoices`
    - `revalidatePath('/dashboard/invoices')`
    - `revalidatePath('/dashboard/orders/' + invoice.order_id)`
    - `redirect('/dashboard/invoices')`
  </action>
  <verify>`npx tsc --noEmit` passes; migration file exists; invoices.ts exports all three actions</verify>
  <done>AC-1 (createInvoice), AC-2 (duplicate check), AC-5 (markInvoiceSent) server logic satisfied</done>
