<objective>
## Goal
Automatically fire the accounting webhook when an invoice is created and when it's marked as paid. Track sync status per invoice so admins can see what's been synced.

## Output
- `accounting_synced_at` and `accounting_sync_status` columns on invoices table
- `fireAccountingSync(invoiceId, event)` utility in src/lib/accounting.ts
- Called automatically in invoice create and payment-recorded actions
- Invoice detail page shows sync status badge (synced / pending / failed)
- Manual "Sync Now" button on invoice detail for failed/missing syncs
</objective>

<acceptance_criteria>

## AC-1: Sync fires on invoice creation
```gherkin
Given accounting webhook is enabled for the tenant
When a new invoice is created
Then a webhook fires with event="invoice.created" and invoice data
  and accounting_sync_status = 'synced' if successful
  or 'failed' if webhook errors
```

## AC-2: Sync fires on payment
```gherkin
Given an invoice is marked as paid
Then a webhook fires with event="invoice.paid" and invoice + payment data
  and accounting_synced_at updated
```

## AC-3: Sync status visible on invoice detail
```gherkin
Given an admin views an invoice
Then they see a sync status badge: "Synced", "Not synced", or "Sync failed"
  and if failed or not synced, a "Sync Now" button is available
```

## AC-4: Webhook disabled = no sync attempts
```gherkin
Given accounting is disabled in tenant settings
When an invoice is created or paid
Then no webhook is fired and accounting_sync_status remains NULL
```

</acceptance_criteria>
