<objective>
## Goal
Add an Accounting Settings page where admins configure their QuickBooks or Xero integration via an n8n webhook URL. The app fires webhooks to this URL when invoices are created or paid — n8n handles the QB/Xero API interaction.

## Architecture
CRM fires webhook → n8n receives → n8n creates/updates invoice in QB or Xero.
No direct QB/Xero OAuth in this app. Each tenant provides their own n8n webhook URL.
This keeps the CRM decoupled from accounting software specifics.

## Output
- Accounting Settings page at /dashboard/settings/accounting (admin-only)
- Fields: accounting_webhook_url (text input), enabled toggle
- `tenant_settings.accounting` JSONB stores config
- "Test Connection" button fires a test webhook and shows success/fail
- Nav link under Settings
</objective>

<acceptance_criteria>

```gherkin

  and a success message is shown
```

## AC-2: Test connection works
```gherkin
Given an admin has saved a webhook URL
When they click "Test Connection"
Then the app sends a test POST to the URL with event type "test"
  and shows "Connection successful" if 2xx response
  or "Connection failed: [error]" if not
```

## AC-3: Accounting webhook can be disabled
```gherkin
Given an admin has configured a webhook URL
When they toggle "Enabled" to off and save
Then future invoice/payment events do not fire the webhook
  even though the URL is still stored
```
