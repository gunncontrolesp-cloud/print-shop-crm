<objective>
## Goal
Add an expiry date to quotes. Expired quotes cannot be accepted by customers. Admins can see expiry status and extend a quote's expiry if needed.

## Output
- `expires_at` column on quotes table
- Expiry date picker on quote builder (default: 30 days from creation)
- "Expires [date]" badge on quote list and detail pages
- "Expired" status shown when past expiry (portal + admin view)
- Admin can set/extend expiry from quote detail
- Portal blocks approval action when quote is expired
</objective>

<acceptance_criteria>

## AC-1: New quotes get a default expiry
```gherkin
Given an admin creates a new quote
Then expires_at defaults to 30 days from created_at
  and the quote detail shows "Expires [date]" badge
```

## AC-2: Expired quotes show correct status
```gherkin
Given a quote's expires_at is in the past
When any user views the quote detail
Then the quote shows an "Expired" badge
  and the portal Approve button is hidden/disabled
```

## AC-3: Admin can extend expiry
```gherkin
Given an admin is on a quote detail page
When they change the expiry date and save
Then expires_at is updated
  and the Expired badge clears if the new date is in the future
```

## AC-4: Portal shows expiry state correctly
```gherkin
Given a customer views a sent quote in the portal
When the quote is not expired
Then Approve button is visible
When the quote is expired
Then Approve button is replaced with "This quote has expired" message
```

</acceptance_criteria>

<task type="auto">
  <name>Task 1: Migration — add expires_at to quotes</name>
  <files>supabase/migrations/[timestamp]_quote_expiry.sql</files>
  <action>
    ```sql
    ALTER TABLE public.quotes
      ADD COLUMN IF NOT EXISTS expires_at TIMESTAMPTZ;

    -- Backfill: set existing sent/pending quotes to 30 days after created_at
    UPDATE public.quotes
    SET expires_at = created_at + INTERVAL '30 days'
    WHERE expires_at IS NULL AND status IN ('pending', 'sent');
    ```
  </action>
  <verify>Migration file exists</verify>
