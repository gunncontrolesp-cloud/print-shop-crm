## AC-1: Dashboard Shows Live Stats
```gherkin
Given the dashboard is loaded by an authenticated user
When the page renders
Then stat cards display: Total Customers, Open Orders (count of non-delivered orders), Pending Approval (pending count), In Production (printing + finishing count)
And a Recent Orders list shows the last 5 orders with customer name, status badge, total, and date
And an Open Quotes card shows count of draft + sent quotes
And all numbers reflect real data from the database (not hardcoded)
```

## AC-2: Empty State is Graceful
```gherkin
Given a freshly seeded database with no orders or quotes
When the dashboard renders
Then stat cards show 0 counts (not errors or blank space)
And the recent orders list shows an "No recent orders" empty state
And the page does not crash or show TypeScript errors
```

## AC-3: Full Lifecycle Works End-to-End
```gherkin
Given migrations 000001–000004 are applied to Supabase
When a user walks through the complete workflow:
  1. Creates a customer
  2. Builds a quote with at least one line item
  3. Approves the quote (admin)
  4. Converts the quote to an order
  5. Advances the order through at least 2 status stages
Then each step completes without error
And the dashboard reflects the updated state after each action
And no duplicate orders are created if "Convert to Order" is clicked twice
```

    Replace the stub with a real server component that fetches live data from Supabase.

    **Data to fetch (parallel, single Supabase client):**

    ```ts
    // Customers total
    const { count: customerCount } = await supabase
      .from('customers').select('*', { count: 'exact', head: true })

    // Orders grouped by status (single query, process client-side)
    const { data: orders } = await supabase
      .from('orders')
      .select('id, status, total, created_at, customers(name)')
      .order('created_at', { ascending: false })

    // Open quotes (draft + sent)
    const { count: openQuoteCount } = await supabase
      .from('quotes')
      .select('*', { count: 'exact', head: true })
      .in('status', ['draft', 'sent'])
    ```

    **Compute from orders array:**
    - `openOrderCount` = orders where status !== 'delivered'
    - `pendingCount` = orders where status === 'pending'
    - `inProductionCount` = orders where status === 'printing' || status === 'finishing'
    - `recentOrders` = first 5 entries from the orders array

    **Layout:**

    Grid of 4 stat cards (top row):
    ```
    [Total Customers]  [Open Orders]  [Pending Approval]  [In Production]
    ```
    Each card: large number, label, subtle icon or colored accent. Use Tailwind — simple, no chart library.

    Below stat cards:

    Two-column layout (or single column on narrow):
    - Left: "Recent Orders" table — customer name, status badge (reuse color scheme from orders list), total, date, "View →" link
    - Right: "Open Quotes" — count + link to /dashboard/quotes

    Status badge colors (same as orders list page):
    - pending: bg-gray-100 text-gray-600
    - approved: bg-blue-100 text-blue-700
    - printing: bg-yellow-100 text-yellow-700
    - finishing: bg-orange-100 text-orange-700
    - completed: bg-green-100 text-green-700
    - delivered: bg-teal-100 text-teal-700

    Empty state for recent orders: "No orders yet. Create a quote to get started."

    Supabase join type guard: `orders.customers` will be inferred as array by TypeScript.
    Use: `(Array.isArray(o.customers) ? o.customers[0] : o.customers)?.name ?? 'Unknown'`

    Avoid: importing a chart library — plain numbers and badges are sufficient for Phase 1.
    Avoid: separate queries for each stat — fetch orders once and compute counts from the array.
    Avoid: showing financial totals/revenue summaries — that's Phase 5 analytics.
