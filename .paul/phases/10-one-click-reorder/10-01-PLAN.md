<objective>
## Goal
Let customers reorder a past job with one click from the portal. The reorder creates a new draft quote pre-filled with the original job's line items and asset files, and notifies the admin to review.

## Purpose
Repeat customers are the highest-value segment. Friction in reordering (calling, re-explaining specs, re-uploading files) costs repeat business. One-click removes that friction entirely.

## Output
- "Reorder" button on each past order in the customer portal
- `reorderJob(orderId)` server action: creates new quote with original line_items JSONB, links original customer asset files, status = 'pending', notifies admin
- Admin sees new quote in dashboard with "Reorder" tag indicating it's a repeat job
- n8n notified so admin gets email: "Customer X wants to reorder [job description]"
</objective>

<acceptance_criteria>

## AC-2: Reorder creates a pre-filled quote
```gherkin
Given a customer clicks Reorder on a past order
When the action completes
Then a new quote is created with:
  - Same customer_id
  - Same line_items JSONB from original quote
  - status = 'pending'

```

## AC-3: Admin is notified
```gherkin

```

## AC-4: Asset files are linked to the new quote
```gherkin

When a reorder is created
Then those asset files are associated with the new order (copied reference, not duplicated in S3)
```
