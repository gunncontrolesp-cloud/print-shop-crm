<objective>
## Goal
New shop admins can complete a one-screen onboarding form that creates their tenant, links their account, and seeds default pricing config — then lands them in the dashboard.

## Purpose
Without onboarding, new shops have no path from signup to usable product. The middleware (06-01) redirects untenanted staff to `/onboarding`, but that route doesn't exist yet. This plan closes that loop: new admin signs up → confirms shop name + tier → tenant is created → pricing is seeded → dashboard is unlocked.

## Output
- `src/lib/actions/onboarding.ts` — `createTenant` server action (service role: insert tenant + link user + seed pricing_config)
- `src/app/onboarding/page.tsx` — onboarding form (shop name, plan tier)
- `src/app/onboarding/layout.tsx` — minimal layout without nav sidebar
- `src/lib/supabase/middleware.ts` — `/onboarding` guards added (no user → /login; has tenant → /dashboard)
</objective>

    ```ts
      materials: [

        { id: 'glossy', label: 'Glossy', multiplier: 1.2 },
        { id: 'matte', label: 'Matte', multiplier: 1.15 },
        { id: 'recycled', label: 'Recycled', multiplier: 1.1 },
      ],
      finishing: [
        { id: 'none', label: 'None', multiplier: 1.0 },
        { id: 'lamination', label: 'Lamination', multiplier: 1.3 },
        { id: 'uv_coating', label: 'UV Coating', multiplier: 1.25 },
        { id: 'foil', label: 'Foil Stamping', multiplier: 1.5 },
      ],
    }

    export async function createTenant(formData: FormData) {
      const supabase = await createClient()
      const {
        data: { user },
      } = await supabase.auth.getUser()
      if (!user) throw new Error('Not authenticated')

      const shopName = (formData.get('shop_name') as string)?.trim()
      const planTier = formData.get('plan_tier') as string

      if (!shopName) throw new Error('Shop name is required')
      if (!['starter', 'pro', 'premium'].includes(planTier)) {
        throw new Error('Invalid plan tier')
      }

      // All three writes use service role — user has no tenant yet so RLS blocks them
      const service = createServiceClient()

      // 1. Create tenant
      const { data: tenant, error: tenantError } = await service
        .from('tenants')
        .insert({ name: shopName, plan_tier: planTier })
        .select('id')
        .single()

      if (tenantError || !tenant) throw new Error(tenantError?.message ?? 'Failed to create tenant')

      // 2. Link user to tenant
      const { error: userError } = await service
        .from('users')
        .update({ tenant_id: tenant.id })
        .eq('id', user.id)

      if (userError) throw new Error(userError.message)

      // 3. Seed default pricing config for this tenant
      const { error: pricingError } = await service
        .from('pricing_config')
        .insert({ tenant_id: tenant.id, config: DEFAULT_PRICING_CONFIG })

      if (pricingError) throw new Error(pricingError.message)

      redirect('/dashboard')
    }
    ```

    Note: `createServiceClient` is already exported from `@/lib/supabase/server`.
    Note: All three DB operations use `service` (not `supabase`) because the user has no tenant_id yet — RLS policies using `get_my_tenant_id()` would block them.
