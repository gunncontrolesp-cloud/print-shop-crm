<objective>
## Goal
Give admins a consolidated view of all files ever uploaded for a customer, with the ability to mark artwork files as "customer assets" — distinct from internal proofs and job files.

## Purpose
Files currently exist only in the context of individual orders. When a customer reorders 18 months later, staff must hunt through old orders to find their artwork. This plan adds a customer-level asset view and an `is_customer_asset` flag to distinguish reusable artwork from one-off job files.

## Output
- `is_customer_asset` boolean column on files table
- "Assets" tab on customer detail page (admin) — aggregates all files for that customer across all orders
- "Mark as Asset" / "Unmark" toggle on each file row
- Files marked as assets surface prominently; others are still accessible but secondary
</objective>

<acceptance_criteria>

## AC-2: Mark/unmark as customer asset
```gherkin

When they click "Mark as Asset" on a file
Then is_customer_asset is set to true
  and the file moves to an "Artwork" section above other files
When they click "Unmark"

```

    ```ts
    export async function toggleCustomerAsset(fileId: string, isAsset: boolean) {
      await requireElevated()
      const supabase = await createClient()
      const { error } = await supabase
        .from('files')
        .update({ is_customer_asset: isAsset })
        .eq('id', fileId)
      if (error) throw new Error(error.message)
      revalidatePath('/dashboard/customers')
    }
    ```
