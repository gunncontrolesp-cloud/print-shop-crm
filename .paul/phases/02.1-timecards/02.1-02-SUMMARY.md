# Phase 2.1 Plan 02: Admin Timecard Dashboard Summary

**Admin can view live clocked-in status, force clock-out employees, approve entries for payroll, and edit times — all shipped and verified working.**

| Criterion | Status | Notes |
|-----------|--------|-------|
| AC-1: Live clocked-in dashboard | Pass | /dashboard/timeclock/admin shows open entries with duration |
| AC-2: Force clock-out | Pass | adminClockOut sets clocked_out_at = now() on open entry |
| AC-3: Approve entries | Pass | adminApproveEntry sets status/approved_by/approved_at |
| AC-4: Edit time entry | Pass | Edit page + adminEditEntryForm updates times + notes, redirects back |

## Accomplishments

- `status`, `approved_by`, `approved_at` columns added to `time_entries` (already applied to Supabase before plan ran)
- `requireAdmin()` helper centralizes auth + role check across all admin actions
- Admin dashboard shows two sections: currently clocked-in (with Force Clock Out) + all closed entries (with Approve + Edit)
- Edit page pre-fills datetime-local inputs from UTC timestamps, submits via FormData server action

## Decisions Made

| Decision | Rationale | Impact |
|----------|-----------|--------|
| `.bind(null, entryId)` for approve/clock-out form actions | Next.js passes bound args first, FormData last — matches `(entryId, _formData?)` signature | Standard pattern for entity-targeted form actions |
| Separate `adminEditEntryForm(formData)` function | Edit form needs to read multiple fields from FormData; can't use bind for multi-field forms | Form submits entryId as hidden input |
| UTC display in edit form (`getUTC*` methods) | Server components can't know user timezone; consistent with how server renders times elsewhere | Times show as UTC in edit form — acceptable for admin corrections |
