<objective>
## Goal
Admin can view a live dashboard of who is clocked in, edit any time entry (correct times/notes), force-clock-out an employee, and approve entries for payroll.

## Purpose
Replaces paper timesheets for the admin — they can reconcile missed clock-outs, correct errors, and mark entries approved before running payroll.

## Output
- Migration adding `status` + `approved_by` + `approved_at` columns to `time_entries`
- Admin server actions: `adminEditEntry`, `adminClockOut`, `adminApproveEntry`
- `/dashboard/timeclock/admin` — live clocked-in view + timecard table with approve buttons
- `/dashboard/timeclock/admin/[id]/edit` — edit form for correcting a single entry
</objective>

<acceptance_criteria>

## AC-1: Live clocked-in dashboard
```gherkin
Given I am an admin on /dashboard/timeclock/admin
When the page loads
Then I see a list of all employees currently clocked in with their clock-in time and elapsed duration
And I see a "Force Clock Out" button for each open entry
```

## AC-2: Admin can force-clock-out an employee
```gherkin
Given an employee has an open time entry
When admin clicks "Force Clock Out" on that entry
Then clocked_out_at is set to now() on the entry
And the entry disappears from the "Currently Clocked In" section
```

## AC-3: Admin can approve time entries
```gherkin
Given a closed time entry has status = 'pending'
When admin clicks "Approve" on that entry
Then entry status changes to 'approved', approved_by = admin user_id, approved_at = now()
And the entry shows an "Approved" badge
```

## AC-4: Admin can edit a time entry
```gherkin
Given any time entry (open or closed)
When admin navigates to its edit page and submits corrected clocked_in_at / clocked_out_at / notes
Then the entry is updated in the database
And admin is redirected back to /dashboard/timeclock/admin
```

## AC-5: Non-admins cannot use admin actions
```gherkin
Given a user with role != 'admin'
When they call adminEditEntry, adminClockOut, or adminApproveEntry
Then a 'Admin access required' error is thrown and no changes are made
```

  <action>
    Add three exported async functions to `src/lib/actions/timeclock.ts`.
    All three must:
    1. Call `createClient()` and verify authenticated user
    2. Fetch `profile` from `users` table — if `profile.role !== 'admin'` throw `new Error('Admin access required')`
    3. Operate and call `revalidatePath('/dashboard/timeclock/admin')`

    **adminClockOut(entryId: string): Promise<void>**
    - Updates `clocked_out_at = now()` on the given entry
    - RLS admin UPDATE policy covers this
    - Throw `'Entry not found'` if no rows updated (check error or rowCount)

    **adminEditEntry(entryId: string, data: { clockedInAt: string; clockedOutAt: string | null; notes: string | null }): Promise<void>**
    - Updates `clocked_in_at`, `clocked_out_at`, `notes` on the entry
    - Do NOT touch `status`, `approved_by`, `approved_at` in this action
    - Validate: clockedInAt must be a valid ISO string; if clockedOutAt provided it must be after clockedInAt

    **adminApproveEntry(entryId: string): Promise<void>**
    - Updates `status = 'approved'`, `approved_by = user.id`, `approved_at = now()`
    - Only approve entries where `clocked_out_at IS NOT NULL` — throw `'Cannot approve open entry'` if entry is still open

    Do NOT modify `clockIn` or `clockOut` — leave employee actions untouched.
  </action>

      'use server'
      const clockedInAt = formData.get('clockedInAt') as string

      const notes = formData.get('notes') as string | null
