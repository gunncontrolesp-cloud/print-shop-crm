## Decisions Made

| Decision | Rationale | Impact |
|----------|-----------|--------|
| Daily cron `0 8 * * *` instead of hourly | Vercel Hobby plan blocks >1 execution/day | Alert fires once daily at 8am — adequate for small shop |
| `npx vercel --prod` for deployment | GitHub auto-deploy integration not triggering on this project | Manual CLI deploy required until GitHub integration fixed |
| Service role client in cron route | No cookie/session context in cron requests | Uses SUPABASE_SERVICE_ROLE_KEY directly |

## Deviations from Plan

| Type | Count | Impact |
|------|-------|--------|
| Auto-fixed | 1 | Cron schedule downgraded from hourly to daily — Hobby plan constraint |
| Deployment issue | 1 | GitHub → Vercel auto-deploy broken; CLI deploy used instead |

**Auto-fix:** Plan specified `"0 * * * *"` (hourly) but Vercel Hobby blocks this. Changed to `"0 8 * * *"` (daily 8am). Upgrade to Vercel Pro ($20/mo) to restore hourly.

**Deployment:** Vercel GitHub integration not triggering on pushes. Workaround: `npx vercel --prod` from terminal. Should be investigated and fixed — currently every deploy requires manual CLI invocation.

## Issues Encountered

| Issue | Resolution |
|-------|------------|
| PowerShell `curl` alias doesn't support `-H` flag | Switched to `Invoke-WebRequest -UseBasicParsing` |

| GitHub → Vercel auto-deploy not triggering | Used `npx vercel --prod` directly |
| Vercel Hobby plan rejects hourly cron | Changed to daily schedule |
