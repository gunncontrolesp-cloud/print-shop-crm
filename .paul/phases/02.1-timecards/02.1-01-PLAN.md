<objective>
## Goal
Add a `time_entries` table with RLS and build the employee clock in/out UI at `/dashboard/timeclock`.

## Purpose
Employees need to self-service clock in and out from a simple, mobile-friendly page. This is the foundation all other timecard features (admin editing, approvals, reports) depend on.

## Output
- Supabase migration for `time_entries` table
- `clockIn` and `clockOut` server actions
- `/dashboard/timeclock` page with clock in/out UI
- Types for `TimeEntry`
</objective>

<acceptance_criteria>

## AC-1: Employee can clock in
```gherkin
Given an authenticated staff user who is not currently clocked in
When they visit /dashboard/timeclock and click "Clock In"
Then a new time_entry row is created with clocked_in_at = now() and clocked_out_at = null
And the page updates to show "Clocked In" status with elapsed time
```

## AC-2: Employee can clock out
```gherkin
Given a staff user who is currently clocked in (open time entry exists)
When they click "Clock Out" on /dashboard/timeclock
Then the open time_entry row is updated with clocked_out_at = now()
And the page updates to show "Clocked Out" status
```

## AC-3: Duplicate clock-in is blocked
```gherkin
Given a staff user who already has an open time entry (clocked_out_at IS NULL)
When clockIn server action is called
Then it returns an error and no new row is inserted
```

## AC-4: RLS isolates tenant data
```gherkin
Given two staff users in different tenants
When each queries time_entries
Then each sees only their own tenant's entries
```

<task type="auto">
  <name>Task 1: Migration — time_entries table with RLS</name>
  <files>supabase/migrations/20260402000001_timecards.sql</files>
  <action>
    Create migration file with:

    1. `time_entries` table:
       - `id` uuid PRIMARY KEY DEFAULT gen_random_uuid()
       - `tenant_id` uuid NOT NULL REFERENCES tenants(id) ON DELETE CASCADE
       - `user_id` uuid NOT NULL REFERENCES users(id) ON DELETE CASCADE
       - `clocked_in_at` timestamptz NOT NULL DEFAULT now()
       - `clocked_out_at` timestamptz NULL  (NULL = currently clocked in)
       - `job_id` uuid NULL REFERENCES jobs(id) ON DELETE SET NULL
       - `task_stage` text NULL
       - `output_qty` integer NULL CHECK (output_qty >= 0)
       - `notes` text NULL
       - `created_at` timestamptz NOT NULL DEFAULT now()

    2. Enable RLS on time_entries

    3. RLS policies (follow exact pattern from 20260327000007_multi_tenant.sql):
       - "Staff can view own tenant time entries" → SELECT WHERE tenant_id = get_my_tenant_id() AND is_authenticated()
       - "Staff can insert own time entries" → INSERT WITH CHECK tenant_id = get_my_tenant_id() AND user_id = auth.uid() AND is_authenticated()
       - "Staff can update own open time entries" → UPDATE WHERE user_id = auth.uid() AND tenant_id = get_my_tenant_id() AND clocked_out_at IS NULL
       - "Admins can update any tenant time entry" → UPDATE WHERE tenant_id = get_my_tenant_id() AND is_admin()
       - "Admins can delete own tenant time entries" → DELETE WHERE tenant_id = get_my_tenant_id() AND is_admin()

    Avoid: creating indexes or triggers — keep migration minimal for now.
  </action>
  <verify>Migration file is valid SQL with no syntax errors. Apply in Supabase dashboard and confirm table appears.</verify>
  <done>AC-4 satisfied: RLS policies isolate by tenant_id</done>
</task>

  <name>Task 2: Types + server actions for clock in/out</name>
  <files>src/lib/types.ts, src/lib/actions/timeclock.ts</files>
  <action>
    **src/lib/types.ts** — append:
    ```ts
    export type TimeEntry = {
      id: string
      tenant_id: string
      user_id: string
      clocked_in_at: string
      clocked_out_at: string | null
      job_id: string | null
      task_stage: string | null
      output_qty: number | null
      notes: string | null
      created_at: string
    }
    ```

    **src/lib/actions/timeclock.ts** — create with 'use server':

    `clockIn(jobId?: string, taskStage?: string)`:
    - Get user from supabase.auth.getUser() — throw if not authenticated
    - Get tenantId from getTenantId()
    - Check for existing open entry: SELECT id FROM time_entries WHERE user_id = user.id AND clocked_out_at IS NULL
    - If open entry exists: throw new Error('Already clocked in')
    - INSERT into time_entries with tenant_id, user_id, clocked_in_at = now(), job_id, task_stage
    - revalidatePath('/dashboard/timeclock')

    `clockOut()`:
    - Get user from supabase.auth.getUser() — throw if not authenticated
    - Find open entry: SELECT id FROM time_entries WHERE user_id = user.id AND clocked_out_at IS NULL ORDER BY clocked_in_at DESC LIMIT 1
    - If none found: throw new Error('Not clocked in')
    - UPDATE time_entries SET clocked_out_at = now() WHERE id = entry.id
    - revalidatePath('/dashboard/timeclock')

    Import getTenantId from '@/lib/tenant'. Use createClient from '@/lib/supabase/server'.
    Avoid: redirect() — let the page handle navigation.
  </action>
  <verify>TypeScript compiles with no errors: npm run build in print-shop-crm directory.</verify>
  <done>AC-1, AC-2, AC-3 satisfied: clockIn blocks duplicates; clockOut closes open entry</done>
</task>
